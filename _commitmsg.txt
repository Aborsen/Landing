Tokenize text colors: gray-200/#8A8A9A -> text-body, #8AA6B3/#5E5E70 -> text-inactive

Replace raw color values and the gray-200 primitive with semantic text tokens
across application code (src/) and the .ins-testimonial__quote component rule:

  var(--ins-color-gray-200) -> var(--ins-text-body)   (41 in src + 1 in components.css)
  #8A8A9A                   -> var(--ins-text-body)    (1)
  #8AA6B3                   -> var(--ins-text-inactive) (84)
  #5E5E70                   -> var(--ins-text-inactive) (1)

Notes:
- gray-200 -> text-body is a no-op color (both resolve to #A8BFC8); it just moves
  usages from the primitive to the semantic token. text-body===gray-200.
- #8AA6B3 (rgb 138,166,179) -> text-inactive (#7FA0AC) slightly darkens muted
  captions/labels; #8A8A9A and #5E5E70 shift to the blue-grays. All intended.
- Left untouched: the token DEFINITIONS in tokens.css (where text-body/icon tokens
  legitimately reference gray-200) and the DS docs.
- Kept 6 donut chart-DATA colors as raw #8AA6B3 on purpose: the chart builds faded
  segments via `color + '55'`, so a var() there would yield invalid CSS
  (var(...)55). #8AA6B3 + '55' = #8AA6B355 stays valid.

Verified in-browser: converted captions compute to rgb(127,160,172); 0 elements
remain on the old rgb(138,166,179); no invalid var()-concat anywhere. Build 31/31.
